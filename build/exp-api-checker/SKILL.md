# Experimental API Checker

Detect experimental API usage in Windows App SDK sample code and generate fix suggestions.

## Features

1. **Detect exp-only API**: Scan C#/C++ code to find experimental-only API usage
2. **Generate fix suggestions**: Analyze code context and suggest how to wrap exp code with macros

## Use Cases

- Before submitting a new sample, check if exp macros are needed
- After CI failure, get detailed fix suggestions  
- After new package release, scan for macros that can be removed

## Commands

### Detect exp API usage

```powershell
./scripts/Detect-ExpApi.ps1 -Path <folder-or-file>
```

Output example:
```
[EXP] Models/LanguageModelModel.cs
      Line 12: using Microsoft.Windows.AI.Text.Experimental;
      Line 45: var model = await LanguageModel.CreateAsync();
      
Suggestion: This file uses exp-only API, wrap with #if WASDK_EXPERIMENTAL
```

### Generate fix code

Call LLM to analyze code and generate specific diff.

## Exp-Only API List

Source: `scripts/exp-only-apis.json`

Generated by `build/winmd-tools/Compare-WinAppSdkApis.ps1`, comparing stable and exp package WinMD metadata.

### Version Strategy

**Always use the latest released versions** (do NOT read from sample csproj):
- Samples may lag behind, but exp-only judgment must follow the latest releases
- After a new stable release, some APIs "graduate" from exp to stable â†’ exp-only list shrinks â†’ exp macros in old samples can be removed
- After a new exp release, new APIs appear â†’ exp-only list grows â†’ new samples can be checked

To refresh the list:

```powershell
# 1. Get latest versions from NuGet
.\Get-LatestVersions.ps1

# 2. Regenerate the API diff
.\Compare-WinAppSdkApis.ps1 -StableVersion "<latest-stable>" -ExperimentalVersion "<latest-exp>"

# 3. Copy the updated list to the checker
Copy-Item ..\winmd-tools\output\exp-only-apis.json ..\exp-api-checker\scripts\
```

The JSON now records `StableVersion` and `ExperimentalVersion` so you can always tell which release pair it corresponds to.

### Main Exp-Only Namespaces

- `Microsoft.Windows.AI.Imaging` - AI image processing
- `Microsoft.Windows.AI.Text.Experimental` - AI text/language models
- `Microsoft.Windows.AI.Video` - AI video processing
- `Microsoft.Windows.Search.AppContentIndex` - App content search
- `Microsoft.UI.Composition.Experimental` - Experimental Composition API
- `Microsoft.UI.Input.Experimental` - Experimental Input API

## Fix Patterns

### Repo Convention

- **Macro name: `WASDK_EXPERIMENTAL`** (defined in `Samples/Directory.Build.props` when `WindowsAppSDKExperimental=true`)
- Stable build promotes `CS8305` ("experimental API") to an error, so unguarded usage fails the build
- Per-sample opt-in via `<WindowsAppSDKExperimental>true</WindowsAppSDKExperimental>` in its own `Directory.Build.props`

### Pattern 1: Hybrid Sample - internal usage

```csharp
#if WASDK_EXPERIMENTAL
using Microsoft.Windows.AI.Text.Experimental;
#endif

internal class LanguageModelModel
{
#if WASDK_EXPERIMENTAL
    private LanguageModelExperimental? _languageModelExperimental;
#endif

    public async Task InitAsync()
    {
        // stable code...

#if WASDK_EXPERIMENTAL
        _languageModelExperimental = new LanguageModelExperimental(_session);
#endif
    }
}
```

### Pattern 2: Hybrid Sample - public method with callers

Use `#if/#else` so the method signature is **preserved** in stable builds; callers do not need to change.

```csharp
#if WASDK_EXPERIMENTAL
public IAsyncOperationWithProgress<Result, string>
GenerateResponseWithLoraAdapter(string contextPrompt, string prompt, string filePath)
{
    // real exp implementation
}
#else
public IAsyncOperationWithProgress<Result, string>
GenerateResponseWithLoraAdapter(string contextPrompt, string prompt, string filePath)
    => throw new NotSupportedException("LoRA adapter support requires the experimental Windows App SDK.");
#endif
```

### Pattern 3: Exp-Only Sample (entire sample requires exp)

Set the flag permanently in the sample's `Directory.Build.props`:

```xml
<Project>
  <Import Project="$([MSBuild]::GetPathOfFileAbove('Directory.Build.props', '$(MSBuildThisFileDirectory)../'))" />
  <PropertyGroup>
    <WindowsAppSDKExperimental>true</WindowsAppSDKExperimental>
    <NoWarn>$(NoWarn);CS8305</NoWarn>
  </PropertyGroup>
</Project>
```
